## Context

此專案為 Hugo + PaperMod 主題的個人部落格。新增文章的標準流程是透過 `hugo new posts/<name>.md` 建立，使用 TOML front matter，內容以繁體中文撰寫。目標是新增一篇技術筆記，介紹 OpenSpec 這個 Spec-Driven Development 框架。

## Goals / Non-Goals

**Goals:**
- 撰寫一篇結構清晰、內容完整的 OpenSpec 介紹文章
- 涵蓋核心概念、工作流程、使用情境、工具比較
- 讓從未接觸過 OpenSpec 的開發者能快速理解其價值

**Non-Goals:**
- 不做 step-by-step 安裝教學（留給官方文件）
- 不深入 OpenSpec 的原始碼分析
- 不製作封面圖片或自訂圖表

## Decisions

### 文章結構設計
採用「問題 → 方案 → 如何運作 → 實際應用 → 比較」的敘事結構，讓讀者從痛點出發，逐步理解 OpenSpec 的價值。

**替代方案**：按官方文件結構重新整理 → 太像翻譯文件，缺乏觀點
**選擇理由**：以使用者視角出發更容易引起共鳴

### 文章檔名
使用 `openspec-spec-driven-development.md`，兼顧 SEO 與可讀性。

### 程式碼範例策略
包含 CLI 指令範例但不包含完整程式碼片段，著重展示工作流程而非實作細節。

**選擇理由**：目標是概念介紹而非教學，過多程式碼會分散焦點

## Risks / Trade-offs

- [OpenSpec 版本迭代快速] → 文章中避免綁定特定版本號，以概念為主
- [資訊量過多導致文章冗長] → 控制在 2000-3000 字，善用標題與列表提升可讀性
- [技術深度與可讀性平衡] → 核心概念用白話解釋，進階功能以連結方式指向官方文件
