## Context

部落格使用 Hugo + PaperMod 主題，所有文章以 markdown 格式存放於 `content/posts/`。現有文章涵蓋 MySQL、認證技術、AI 工具等主題，但尚無 Go 語言相關系列。此系列將是部落格首個完整的程式語言教學系列。

讀者背景假設：有基本程式設計經驗（如 Python、JavaScript），但未接觸過 Go。

## Goals / Non-Goals

**Goals:**
- 建立一套由淺入深的 Go 語言教學系列，讓讀者能從零開始學習
- 最終引導讀者完成一個具備 CRUD、認證、資料庫整合的完整 Web 應用
- 每篇文章獨立可讀，但整體串聯成完整學習路徑
- 所有範例程式碼可直接執行，讀者能跟著動手做

**Non-Goals:**
- 不涵蓋 Go 的進階主題（如 CGO、unsafe、compiler internals）
- 不建構生產等級的部署流程（CI/CD、Kubernetes）
- 不深入前端框架整合（僅使用簡單 HTML template 或 JSON API）
- 不涵蓋微服務架構

## Decisions

### 1. 系列文章結構：分兩大篇章

**選擇**: 基礎篇（5 篇）+ 實戰篇（5 篇），共 10 篇
**理由**: 基礎篇讓完全沒接觸過 Go 的讀者打好根基，實戰篇從 HTTP 開始逐步建構完整應用。比起一口氣混合概念更容易消化。
**替代方案**:
- 只寫 5 篇快速入門 → 太淺，讀者學不到足夠知識去建構應用
- 寫 15+ 篇深入每個主題 → 太長，讀者容易中途放棄

### 2. Web 框架選擇：使用標準庫 `net/http` + 輕量路由器

**選擇**: 以 Go 標準庫為主，搭配 `chi` 或 Go 1.22+ 的增強 `http.ServeMux`
**理由**: 體現 Go 「少即是多」的哲學，讓讀者理解底層原理而非依賴框架黑魔法。標準庫在 Go 1.22 後大幅增強路由功能。
**替代方案**:
- Gin → 最受歡迎但隱藏太多底層細節，不利教學
- Echo → 類似 Gin 的問題
- Fiber → 基於 fasthttp，偏離標準庫太遠

### 3. 資料庫：使用 SQLite + `database/sql`

**選擇**: SQLite 搭配標準 `database/sql` 介面
**理由**: 零配置、無需額外安裝，降低讀者跟著做的門檻。`database/sql` 是 Go 的標準資料庫介面，學會後可輕鬆切換 PostgreSQL 等。
**替代方案**:
- PostgreSQL → 需要額外安裝與設定，增加跟做難度
- GORM → ORM 隱藏太多 SQL 細節

### 4. 文章命名規則

**選擇**: `golang-series-XX-topic.md` 格式（如 `golang-series-01-introduction.md`）
**理由**: 數字前綴讓系列順序一目瞭然，讀者在部落格中容易識別系列文章。

### 5. 文章語言與程式碼

**選擇**: 文章以繁體中文撰寫，程式碼與變數名稱保持英文
**理由**: 符合部落格的既有語言風格與讀者群體，技術術語保留英文以利查找官方文件。

## Risks / Trade-offs

- **Go 版本更新** → 以 Go 1.22+ 為基準撰寫，明確標註版本要求。標準庫為主的策略降低第三方套件版本問題。
- **系列長度與讀者留存** → 每篇控制在 15-20 分鐘閱讀量，確保資訊密度適中。文章可獨立閱讀降低必須從頭開始的壓力。
- **範例程式碼可能過時** → 使用標準庫為主，減少外部依賴。在文章開頭標註使用的 Go 版本。
